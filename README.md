# aAI Annotation & LLM Evaluation

This project supports automatic annotation of Italian autobiographical narratives using the **Adapted Autobiographical Interview (aAI)** scoring schema, and evaluates LLM performance against human gold annotations.

The aAI schema classifies text spans as *Internal* details (events, places, times, perceptual details, emotions/thoughts) or *External* details (general semantics, personal semantics, episodic/generic events, repetitions, other) relative to the main described event. See [guidelines.md](guidelines.md) for the full scoring manual.

## Data

Transcripts are Italian descriptions of autobiographical events, stored as [WebAnno](https://webanno.github.io/webanno/) TSV files under `annotation/`. Each subfolder (e.g. `P1_O1.txt`) corresponds to one transcript: `P<participant>_<O|W><session>` where `O` = oral and `W` = written.

The dataset is split into a train set (20 files) and a test set (20 files), defined in `call-llm.py` and `save-txt.py`.

## Setup

```bash
pip install -r requirements.txt
```

Copy `.env.example` to `.env` (or edit `.env` directly) and set:

```
OPENAI_API_KEY=your_key_here
OPENAI_MODEL=gpt-4o            # or any OpenAI-compatible model
OPENAI_BASE_URL=               # optional: custom base URL for non-OpenAI endpoints
```

## Scripts

### 1. `save-txt.py` — Export plain text and gold annotations

Reads the WebAnno TSV files and writes:
- `txt_files/<name>` — plain text of each transcript
- `gold_files/<name>` — gold annotations as JSON (`[{"text": "...", "type": "..."}]`)

```bash
python save-txt.py
```

### 2. `stratification.py` — Stratified train/test split

Computes a balanced 50/50 train/test split using iterative multi-label stratification over annotation label distributions and oral/written register. Prints the resulting split and per-label counts.

```bash
python stratification.py
```

### 3. `call-llm.py` — Run LLM annotation

Sends each test transcript to an OpenAI-compatible LLM with either a **few-shot** prompt (using all train files as examples) or a **zero-shot** prompt. Outputs are saved under `llm_outputs/<model>/<few_shot|zero_shot>/`.

Set `KIND = "few_shot"` or `KIND = "zero_shot"` at the top of the file.

```bash
python call-llm.py
```

Skips files that already have output. Saves the rendered prompt alongside each output as `<name>.prompt.md`.

### 4. `eval-llm.py` — Evaluate predictions against gold

Computes precision, recall, and F1 using **character-level overlap** between predicted and gold spans, per label and micro-averaged.

**Single file:**
```bash
python eval-llm.py \
  --gold gold_files/P2_W2.txt \
  --pred llm_outputs/gpt-4o/few_shot/P2_W2.txt \
  --text-file txt_files/P2_W2.txt
```

**Batch (entire folder):**
```bash
python eval-llm.py \
  --input-folder llm_outputs/gpt-4o/few_shot \
  --gold-folder gold_files \
  --text-folder txt_files
```

Optional flags:
- `--case-sensitive` — enable case-sensitive matching (default: case-insensitive)
- `--keep-non-alnum` — keep punctuation when matching (default: stripped)
- `--show-details` — print unresolved or ambiguous spans

### 5. `iaa.py` — Inter-annotator agreement

Computes **Krippendorff's alpha** with Jaccard distance over token sets, globally and per label, comparing the two annotators (`beatricebozzetto.tsv` and `martinavacondio.tsv`).

```bash
python iaa.py
```

### 6. `count.py` — Annotation statistics

Counts annotation instances per file and label, and exports the table to `stats_2.xlsx`.

```bash
python count.py
```

## Project structure

```
annotation/          WebAnno TSV files (one folder per transcript)
txt_files/           Plain text exports (generated by save-txt.py)
gold_files/          Gold JSON annotations (generated by save-txt.py)
llm_outputs/         LLM predictions (generated by call-llm.py)
guidelines.md        aAI scoring manual
prompt.md            Few-shot LLM prompt template
prompt-zero.md       Zero-shot LLM prompt template
single-example.md    Template for a single few-shot example
```
